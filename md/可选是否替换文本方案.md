# 可选是否替换文本方案

## 解决的问题

当前交互模式在预览阶段只有一个全局 Y/n 确认：

```
预览 → 用户查看 → 确认替换？[Y/n]
```

用户只能"全部替换"或"全部不替换"。但实际场景中，**大部分条目需要替换，极少部分不想替换**（比如 `form.status = '已通过'` 这种数据值）。用户需要一个机制在预览阶段排除特定条目。

---

## 可行性评估

### 架构契合度：高

当前交互流程（`runInteractiveFlow`）已经是分步确认的：

```
配置 → 初始化 → 预览 → AI翻译 → 替换
```

改造点是把预览后的"全局 Y/n"升级为"逐项勾选"，不改变核心解析和替换模块。

### 数据流支持：完全支持

```
prepareScanResults()  →  预览展示  →  用户勾选  →  runScan() 只替换勾选项
     ↓                                                    ↑
  matched[]                                          过滤掉被排除的
  unmatched[]
  special[]
```

`replaceInFile()` 本身就是逐 item 处理的，传入的 items 数组过滤一下即可，**不改 replacer 核心逻辑**。

### 终端 UI 轮子：已有

`setup.cjs` 里已经实现了完整的多选交互：

- 方向键 `↑↓` 移动光标
- `Space` 切换选中/取消
- `Enter` 确认
- raw mode 逐键监听

可以直接复用这套逻辑。

### 结论：可行

改动集中在 `index.cjs` 的交互流程部分，核心解析和替换模块不用动。

---

## 设计方案

### 核心思路：默认全选，按需排除

用户只需翻到不想改的那几条，按 `Space` 取消勾选，其余全部替换。这符合"大部分要改、极少部分不想改"的使用场景。

### 交互流程

```
预览展示（按文件汇总）
  │
  ├─ src/views/list.vue: 12 条已匹配, 3 条未匹配
  ├─ src/views/start.vue: 8 条已匹配
  └─ ...
  │
  ▼ 用户选择
  ┌─────────────────────────────────────────────┐
  │ 默认全部替换，是否需要排除某些条目？ [y/N]    │
  │                                             │
  │ （选 N → 直接全部替换，和现在一样）           │
  │ （选 Y → 进入逐文件浏览模式）                 │
  └─────────────────────────────────────────────┘
  │
  ▼ 逐文件浏览（只对有匹配项的文件）
  ┌─────────────────────────────────────────────┐
  │ 文件: src/views/list.vue                    │
  │                                             │
  │ ◉ L12  static-attr  label="已通过"           │
  │ ◉ L15  text-content  确认删除               │
  │ ◉ L23  script-string '加载失败'              │
  │ ◯ L30  script-string '已通过'  ← 用户取消    │
  │                                             │
  │ ↑↓ 移动  Space 排除/保留  Enter 确认         │
  └─────────────────────────────────────────────┘
```

默认全部勾选（`◉`），用户按 `Space` 取消不想替换的条目（`◯`），Enter 确认后进入下一个文件。

### 条目唯一标识

沿用现有的去重 key：`${file}:${line}:${chineseText}`

这个组合在 `prepareScanResults` 的去重阶段已经存在，可以直接用作条目 ID。

### 排除项持久化

用户排除的条目写入 `.i18n-ignore.json`（项目根目录），后续扫描自动跳过：

```json
{
  "excluded": [
    "src/views/list.vue:30:已通过",
    "src/views/start.vue:45:收入"
  ]
}
```

好处：
- 下次扫描自动跳过，不需要反复勾选
- 可以手动编辑，批量添加排除规则
- 可以提交到 git，团队共享排除配置

## 实现计划

### 改动点

| 文件 | 改动 | 说明 |
|------|------|------|
| `index.cjs` | 新增排除交互环节 | 在 `runInteractiveFlow` 预览步骤后插入 |
| `index.cjs` | 新增多选 UI 函数 | 复用 `setup.cjs` 的 `rawListen` + `renderMultiOptions` 模式 |
| `index.cjs` | `runScan()` 增加过滤参数 | 传入 `excludeIds`，过滤 items |
| `index.cjs` | `.i18n-ignore.json` 读写 | `prepareScanResults` 加载，排除环节写入 |
| 其他模块 | 不改动 | scanner / replacer / translator 均不受影响 |

### 实现步骤

1. **加载已有排除项**：`prepareScanResults` 中读取 `.i18n-ignore.json`，过滤掉已排除的条目
2. **预览展示**：和现在一样，按文件汇总展示
3. **排除交互**：用户选择进入排除模式后，逐文件展示条目，默认全选，Space 取消
4. **收集排除 ID**：用户取消勾选的条目 ID 写入 `.i18n-ignore.json`
5. **执行替换**：`runScan()` 只处理未被排除的条目

### 边界情况

| 情况 | 处理 |
|------|------|
| 条目数量很大（100+） | 按文件分组浏览，每个文件的条目数通常可控 |
| 用户中途退出（Ctrl+C） | 已排除的条目不保存，下次扫描重新来 |
| `.i18n-ignore.json` 中的条目已不存在 | 扫描时忽略不存在的条目，不报错 |
| 非交互模式（`-s` / `-a`） | 不进入排除环节，全部替换，和现在一样 |
