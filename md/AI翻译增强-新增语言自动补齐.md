# AI 翻译增强：新增语言自动补齐

## 背景

现有 `--translate` 命令只能处理**新扫描到的中文文本**，无法处理**新增目标语言后已有文本的翻译缺口**。

**典型场景：** 项目已有 zh-CN + en 双语言包（100+ 条），现在要新增泰语（th）。旧方案需要手动补齐 100+ 条泰语翻译，新方案一键自动完成。

## 实现方案

在现有翻译流程中插入**缺口检测**和**缺口补齐**两个环节：

```
Step 1:   扫描 Vue 文件中的中文 → 去重
Step 2:   加载已有语言包 → 过滤已翻译 → 检测翻译缺口
Step 3a:  AI 翻译新增文本（生成 key + 翻译）
Step 3b:  AI 补齐缺口（复用已有 key，只补缺失语言）
Step 4:   一次性写回所有语言包
```

### 核心逻辑

- **缺口检测**：遍历 zh-CN.json 所有条目，逐条检查各目标语言文件是否存在对应翻译，缺失的归入缺口列表
- **缺口补齐**：按缺失语言组合分组（如「只缺 th」和「缺 en+th」分开），使用填空式 JSON 模板调用 AI，降低出错概率
- **结果校验**：AI 返回后逐条校验完整性，缺失项自动重试（最多 3 次），仍有缺失的记入日志

## 亮点

### 1. 填空式 AI 调用

传统做法是让 AI 自由输出，容易产生格式偏差。本方案给 AI 一个「带空位的 JSON 模板」，AI 只需填空调，大幅降低解析失败率：

```json
// 输入（模板）
{ "transfer.sponsor": { "th": "" } }

// 输出（AI 填空）
{ "transfer.sponsor": { "th": "ผู้สนับสนุน" } }
```

### 2. 按语言组合智能分组

不同条目的缺失语言可能不同（有的缺 th，有的缺 en+th），按组合分组后分批请求，每批的 prompt 更精准，翻译质量更高。

### 3. 已有翻译零覆盖

缺口补齐只写缺失的语言，已有翻译（如 en.json）不受任何影响，不会出现「跑一遍脚本把之前的翻译改坏了」的情况。

### 4. 参考语言包完整性校验

加载参考语言包时进行三重阻塞性校验（路径存在 → 文件齐全 → key 数量一致），不通过则直接报错退出，防止用户误以为参考包已生效。

### 5. 空项目自然兼容

全新项目（零语言包）启动时，缺口检测结果为空，所有文本走新增翻译路径，无需特殊处理。

### 6. API 响应格式约束

请求参数中加入 `response_format: { type: "json_object" }`，在 API 层面强制模型输出合法 JSON，与 prompt 文字约束形成双重保障。

## 效果验证

以新增泰语为例，实际运行结果：

```
[2/4] 加载已有翻译...
  本项目语言包: 96 条
  翻译缺口: 96 条（已有 key 但缺少目标语言翻译）

[3b/4] 补齐翻译缺口...
  缺口组 [th]: 96 条，分 1 批
    ✓ 完成

==================================================
AI 翻译完成
  新增翻译:     0
  缺口补齐:     96
  失败批次:     0
==================================================
```

zh-CN / en / th 三个语言包均为 96 条，结构完全一致，en.json 已有翻译未被修改。
