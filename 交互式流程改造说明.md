# toI18n 交互式全流程改造

## 改动概述

将 `node toI18n.cjs`（无参数）从"仅运行配置向导"改为**逐步确认的完整引导流程**，同时保留所有分步 CLI 模式供本地调试。

## 改动文件

**`scripts/i18n-scan/index.cjs`** — 唯一改动文件

## 具体改动

### 1. 新增 `confirm()` 辅助函数

简单的 readline Y/n 确认提示，用于流程中各步骤之间的用户确认。setup.cjs 的 readline 在 `runSetup()` 的 `finally` 中已关闭，不会冲突。

### 2. 新增 `runInteractiveFlow()` — 交互式全流程

无参数执行 `node toI18n.cjs` 时的默认行为：

```
Step 0: 加载/创建配置
  ├─ 已有配置 → "检测到已有配置，是否修改？" [y/N]
  └─ 无配置 → 进入配置向导

Step 1: 初始化 → "是否继续？" [Y/n]

Step 2: 扫描预览 → 输出 dry-run 结果 → "是否继续？" [Y/n]
  └─ 0 条中文 → 提前结束

Step 3: AI 翻译（仅 ai.enabled 时）
  ├─ 无 API Key → 跳过提示
  └─ "是否进行 AI 翻译？" [Y/n] → "是否继续？" [Y/n]

Step 4: 执行替换 → "是否执行？" [Y/n]
  ├─ Y → 替换源文件 + 更新语言包
  └─ N → 退出
```

每步有 Y/n 确认，用户可随时停下来查看输出，选 N 则优雅退出。

### 3. 提取 `prepareScanResults()` 函数

将扫描 + 分类 + 分组逻辑从 `runScanMode()` 中提取为独立函数，供 `runScanMode` 和 `runInteractiveFlow` 复用。

### 4. 修复 `--all` 模式 bug

`runScanMode(config)` 缺少 modeOverride 参数，导致第 3 步（扫描替换）实际不执行。改为 `runScanMode(config, 'scan')`。

### 5. `runScan()` 支持 `skipConfirm` 选项

新增 `options.skipConfirm` 参数：
- CLI `--scan` 模式：`{ skipConfirm: true }` — 保持原有行为（不弹确认）
- 交互式流程：`{ skipConfirm: true }` — 已在外部确认过，不重复提示

## 构建

```bash
npm run build   # 输出 toI18n.cjs（~3.3MB，包含所有依赖）
```

构建脚本 `scripts/i18n-scan/build.cjs` 使用 esbuild，输出格式为 CommonJS（`.cjs`），用户无需 `npm install`，拿到文件即可运行。

## 使用方式

| 场景 | 命令 | 说明 |
|------|------|------|
| 用户首次使用 | `node toI18n.cjs` | 交互式全流程，逐步确认 |
| CI / 无人值守 | `node toI18n.cjs --all` | 非交互全流程，不暂停 |
| 仅预览 | `node toI18n.cjs --dry-run` | 不修改文件 |
| 仅替换 | `node toI18n.cjs --scan` | 扫描并替换 |
| 仅翻译 | `node toI18n.cjs --translate` | AI 翻译 |
| 盲区扫描 | `node toI18n.cjs --gap` | 输出所有中文 |
| 仅初始化 | `node toI18n.cjs --init` | 创建 locale 目录结构 |
| 本地分步调试 | `node scripts/i18n-scan/index.cjs --scan` | 使用源码直接运行 |
